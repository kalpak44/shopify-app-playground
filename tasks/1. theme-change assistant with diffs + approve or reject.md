You are extending an existing Shopify Remix app.

Goal:
Build an MVP “AI Theme Change Assistant” inside the Shopify embedded app.

The merchant should chat with the app and request theme changes. The app should inspect the active Shopify theme, propose safe file changes, show diffs, and let the merchant approve or reject before applying changes.

Do not implement AI provider/model configuration yet.
Do not implement generated storefront blocks yet.
Do not implement billing.
Focus only on safe theme-change workflow.

==================================================
CORE USER FLOW
==================================================

1. Merchant opens the embedded app.
2. Merchant goes to “Theme Assistant”.
3. Merchant writes a request, for example:
   “Add a sale banner to the homepage”
4. App reads the active theme files.
5. App creates a proposed change.
6. App shows:
    - files that would change
    - before/after diff
    - short explanation
7. Merchant can:
    - Approve and apply
    - Reject
8. If approved, app writes changes to the theme.
9. App stores the change history.

==================================================
IMPORTANT SHOPIFY REQUIREMENTS
==================================================

Use Shopify Admin API theme file APIs.

The app needs theme access permissions, likely:
- read_themes
- write_themes

Before writing theme files:
- detect the active MAIN theme
- read target files
- create a backup record in DB
- show diff
- require explicit merchant approval

Do not automatically modify theme files from chat without approval.

==================================================
DATABASE MODELS
==================================================

Add Prisma models:

model ThemeChangeSession {
id        String   @id @default(cuid())
shop      String
title     String?
status    String   @default("open")
createdAt DateTime @default(now())
updatedAt DateTime @updatedAt

messages  ThemeChangeMessage[]
proposals ThemeChangeProposal[]

@@index([shop])
}

model ThemeChangeMessage {
id        String   @id @default(cuid())
sessionId String
role      String
content   String
createdAt DateTime @default(now())

session ThemeChangeSession @relation(fields: [sessionId], references: [id], onDelete: Cascade)

@@index([sessionId])
}

model ThemeChangeProposal {
id          String   @id @default(cuid())
sessionId   String
shop        String
themeId     String
status      String   @default("pending")
summary     String
files       Json
createdAt   DateTime @default(now())
updatedAt   DateTime @updatedAt

session ThemeChangeSession @relation(fields: [sessionId], references: [id], onDelete: Cascade)

@@index([shop])
@@index([sessionId])
@@index([shop, status])
}

The proposal files JSON should contain:

[
{
"path": "templates/index.json",
"before": "...",
"after": "...",
"diff": "..."
}
]

==================================================
DIFFS
==================================================

Generate a unified diff for each file.

Use a JS diff library if available, or implement a simple line-based unified diff.

Display:
- file path
- summary
- before/after diff

Do not write changes during proposal generation.

Only write after approval.

==================================================
APPROVAL LOGIC
==================================================

Approve proposal:
- verify proposal.shop matches current session.shop
- verify proposal.status is pending
- for each file:
    - re-read current theme file
    - check it still matches the stored "before"
    - if mismatch, stop and show conflict error
    - write stored "after"
- mark proposal as approved
- add assistant message: "Change applied."

Reject proposal:
- mark proposal as rejected
- add assistant message: "Change rejected."

==================================================
SAFETY RULES
==================================================

Strictly follow these rules:

- Never apply a theme change without approval.
- Never trust shop/theme/session IDs from user input without checking ownership.
- Only use session.shop from Shopify auth.
- Only support templates/index.json for MVP.
- Only generate JSON template changes.
- Do not generate Liquid.
- Do not generate JavaScript.
- Do not generate CSS.
- Do not edit live theme files until approval.
- On approval, check that the theme file has not changed since proposal creation.
- If the file changed, show conflict and do not apply.
